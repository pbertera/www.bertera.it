Running rootless Red Hat Quay with podman

Posted on Wed 23 February 2022 in Systems

Podman, as the name suggests (Pod Manager tool) has the concept of pods.

Basically a pod is a goup of containers sharing the same kernel namespaces. When a pod is created an "infra" container is instantiated with the purpose to hold the namespaces associated with the pod.

The other containers belonging to the pod are spawn using the same namespaces.

This feature comes handy when you have to deploy applications composed by multiple containers.

Icing on the cake is that podman supports the Kubernets Pod YAML syntax, thus you can import and export the workloads easily.

Setting up Quay Container Registry

First of all we have to create three folders where the permanent data is stored:

  • ~/quay/config contains the congifuration
  • ~/quay/postgres contains the pg-data of the PosgreSQL container
  • ~/quay/storage holds the container images served by quay
$ mkdir -p ~/quay/{config,postgres,storage}

Since the PostgreSQL container is running with UID 26 we have to set an ACL for the postgres folder

$ setfacl -m u:26:-wx ~/quay/postgres
